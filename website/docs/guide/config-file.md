# 配置文件

手势与相关偏好以 **JSON** 持久化。日常推荐用界面**导入 / 导出**部分手势；整库备份可直接复制本地文件。

## 界面导入 / 导出

**设置 → 手势**：

1. **导出**：多选手势 → 导出为 JSON 包（格式与配置版本字段一致）
2. **导入**：选择 JSON 文件；若与现有手势内容重复，可**跳过重复**或**强制导入**（强制导入的重复项默认关闭）

适合分享几条常用手势、在多台机器间同步子集，而无需覆盖整个配置库。

## 整库路径

```text
~/Library/Application Support/StrokeMouse/gestures.json
```

设置界面可「在 Finder 中显示」该目录。

## 使用建议

| 场景 | 做法 |
|------|------|
| 分享 / 同步部分手势 | **设置 → 手势 → 导出 / 导入** |
| 整库备份 | 复制整个 `StrokeMouse` 目录或单独复制 `gestures.json` |
| 换机 | 装好 App 并授权后：导入 JSON 包，或覆盖同名 `gestures.json` 再启动 |
| 手工编辑 | 保持合法 JSON；未知字段应尽量向后兼容（缺省值 / 可选） |
| 损坏恢复 | 可删掉文件让应用重新生成默认配置（会丢失自定义） |

::: warning
应用运行时可能写回配置。请在退出 App 或确认无并发写入时再覆盖整库文件。
:::

## 配置里大致有什么

每条手势（profile）通常包含：

- 唯一 id、名称、是否启用
- **触发键**（右 / 中 / 侧键等）
- **轨迹**（自由路径点列；旧版方向序列仍可解码并转换）
- **动作**（快捷键、App、URL、媒体、窗口、Shell、AppleScript 等）
- **作用域**（全局或 bundle id 列表）
- 备注

具体字段以应用版本的 `Codable` 模型为准；升级后旧文件应尽量可读。导入时会迁移旧版方向序列为 free-path。

## 与 UI 的关系

日常请优先用 **设置 → 手势** 编辑与导入导出。整库 JSON 适合：

- 完整备份与换机还原
- 在版本库中保存个人配置（注意勿提交含隐私的脚本）
- 排查损坏的配置

## 隐私

Shell / AppleScript 内容可能含路径或令牌。分享导出包或配置文件前请自行脱敏。
