# 常见问题 FAQ

## 手势完全没反应？

按顺序检查：

1. **辅助功能**是否已授权**当前这份** App（换路径 / 重签名后要重授）
2. 菜单栏是否显示引擎暂停或权限错误
3. 对应手势是否 **启用**
4. 是否按住了该手势配置的 **触发键**（默认右键，不是左键）
5. 滑动是否过短（短按会回放为普通点击，不触发动作）

详见 [权限说明](./permissions) 与 [快速开始](./getting-started)。

## 右键菜单没了？

短按右键（几乎不拖动）应仍弹出菜单。若整段拖动已超过最小距离，事件会用于手势而**不会**同时出菜单——这是预期行为。

把「需要菜单」的操作改成轻点，把「需要手势」的操作画够长度即可。也可把手势改到**中键 / 侧键**，把右键完全留给系统菜单。

## 总是误触发或匹配错手势？

- 为易混轨迹增加差异（方向、转折）
- 删除不用的候选，减少同触发键下的竞争
- 重新录制更干净的模板
- 用 App 作用域限制只在某些软件生效

## 匹配太严 / 太松？

识别含分数阈值与结构门控。过严：放慢重录、轨迹更接近模板；过松：改模板形状、减少相似手势。

## 窗口动作不生效？

确认辅助功能已开；部分 App 对 Accessibility 支持不佳。可改用等价快捷键动作。

## Shell / AppleScript 没跑起来？

- 命令 / 脚本本身是否在「终端 / 脚本编辑器」可独立运行
- AppleScript 控制其他 App 时是否授予了**自动化**
- 查看是否有系统弹窗被挡住

## 登录后没有自动启动？

在 **设置 → 通用** 打开登录启动。若系统设置里被关过「登录项」，需两边都打开。

## 深色 / 浅色不对？

通用设置中可强制主题，或改回「跟随系统」。

## 配置丢了？

检查 `~/Library/Application Support/StrokeMouse/gestures.json` 是否被删或覆盖。有备份则退出 App 后还原。见 [配置文件](./config-file)。

## 需要什么鼠标？

任意带可配置触发键的鼠标即可。默认使用**右键**；可在手势编辑中改为中键或侧键。触控板点击通常可作为鼠标键使用，完整多指手势不是本产品的使用方式。

## 如何备份配置？

复制 `~/Library/Application Support/StrokeMouse/` 目录，或仅备份 `gestures.json`。换机后装好 App 并授权，再还原文件即可。
