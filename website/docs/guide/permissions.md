# 权限说明

StrokeMouse 需要全局鼠标监听与（可选）自动化能力。权限失败会在**菜单栏状态**和**设置 → 权限**中可见，而不是静默无效。

## 权限一览

| 权限 | 是否必须 | 用途 |
|------|----------|------|
| **辅助功能（Accessibility）** | **必须** | 全局鼠标事件（`CGEventTap`）、快捷键注入、窗口 AX 操作 |
| **自动化（Automation）** | 可选 | 使用 AppleScript 控制其他 App 时按需授权 |

## 辅助功能

### 为什么需要

手势引擎在系统级拦截**已配置触发键**的鼠标序列。没有辅助功能信任时，`AXIsProcessTrusted()` 为否，应用**不得**假装正在监听。

### 如何开启

1. 打开 StrokeMouse **设置 → 权限**
2. 点击跳转到系统设置：**隐私与安全性 → 辅助功能**
3. 打开 StrokeMouse 的开关
4. 如列表中无条目，可先运行一次 App，或按界面提示添加

### 常见情况

- **升级 / 重新签名 / 换路径构建**：系统可能要求重新授权
- **开关已开但无效**：关掉再开，或移除条目后重新添加并重启 App
- **多份 App 副本**：只对你实际启动的那一份授权

## 自动化

仅在动作类型为 **AppleScript** 且脚本要控制其他应用时需要。系统会在首次相关调用时弹出授权对话框，按 App 粒度确认即可。

## 安全原则

- 全局事件监听属于高权限能力，请从可信渠道获取构建产物
- **Shell / AppleScript** 可执行任意本地命令，UI 中会保留风险提示；**只添加你信任的脚本**
- 日志中不会把用户脚本全文打到公开渠道（工程约定）

## 自检清单

若手势完全无反应：

1. 菜单栏是否显示权限 / 引擎异常？
2. 系统辅助功能是否已勾选当前这份 App？
3. 手势是否被暂停？
4. 是否在用**未配置**为触发键的鼠标按钮？
