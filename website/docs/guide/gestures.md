# 手势系统

理解触发键、轨迹采样与匹配规则，才能稳定自定义手势。

## 核心流程

```text
按住触发键 → 采样路径 → 位移 ≥ 最小滑动距离
       → 松开 → 在「同触发键」的候选中匹配 → 执行动作
```

1. **触发键写在每条手势配置上**（默认右键）
2. 引擎只监听**已启用手势**用到的按键集合
3. 松开时结合 **App 作用域** 与前台 `bundleIdentifier` 过滤候选

## 触发键

每条手势可独立选择：

| 触发键 | 说明 |
|--------|------|
| 右键 | 默认；短按仍可出右键菜单 |
| 中键 | 滚轮按下 |
| 侧键后退 / 前进 | 常见鼠标侧键 |

左键与未监控的按钮**始终放行**，不会被手势引擎劫持。

## 短按回放

触发键的 down 由引擎接管后：

- 若拖动距离 **未达到** `minStrokeDistance`：用带标记的合成 down/up **回放**为普通点击，保证右键菜单等行为可用
- 若已开始有效绘制：该组 down/drag/up **只用于手势**，不会同时交给前台 App

## 轨迹匹配（freePath）

当前主路径为**自由轨迹模板匹配**：

- 有序弧长重采样
- 1D / 2D 归一化
- **±12°** 有限旋转搜索
- 分数需 ≥ 配置阈值
- **显著段数 / 连续转角** 作为结构门控（不可被分数补偿）
- **不使用**镜像、逆序或 near-miss 兜底

录制时尽量画干净、速度适中；复杂花体可多录几次选最稳的一条。

## 实时 HUD

按住触发键拖动时，屏幕上会绘制当前轨迹，便于对齐模板方向与形状。

## App 作用域

| 作用域 | 行为 |
|--------|------|
| 全局 | 任意前台 App 都可匹配 |
| 指定 App | 仅当前台 bundle id 落在列表内时参与匹配 |

适合「仅在浏览器里有效」或「仅开发工具里有效」的手势。

## 启用 / 禁用

每条手势可单独开关。禁用后不参与监听集合与匹配，无需删除配置。
